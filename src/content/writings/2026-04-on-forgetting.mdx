Companies need a world model. Jack Dorsey called it that — a machine-readable substrate that holds decisions, discussions, and code as something more than an archive, replacing the slow human work of routing context through hierarchy. He's pointing at the right primitive.

But every "AI second brain" being shipped is single-player. Obsidian + Claude. Mem0. Letta. Supermemory. They all benchmark on retrieval, treating memory as storage with smarter lookup. None of them touch the part of memory that actually matters.

Memory's hard problem isn't adding things. It isn't even retrieving them. It's choosing what stops mattering — and that work only becomes load-bearing where individuals can't reach.

The cog-sci literature has been clear on this for fifty years. Bjork's storage-vs-retrieval-strength model gave us the language: storage persists, retrieval decays, and decay is what keeps the path clear.<Sidenote>Bjork & Bjork's "new theory of disuse" (1992): storage strength only grows; retrieval strength fluctuates. Forgetting *is* the path to relearning.</Sidenote> Nader's reconsolidation work showed that every retrieval is a potential write. Complementary-learning-systems theory explains why: without supersession, contradictory beliefs accumulate; without schema formation, every episode stays equally weighted. A 2026 ACM Computing Surveys piece on selective forgetting calls it "adaptive, not a defect." Yet every shipping memory product treats forgetting as data loss.

The reason is that forgetting needs a substrate. It only becomes load-bearing where three things appear together:

- **Recurrence** — the same patterns showing up across many sessions, so the system has signal to extract
- **Contradiction** — newer beliefs that supersede older ones, so supersession has work to do
- **Schemas** — abstractions worth keeping, so consolidation has a target

Individuals don't generate enough of any of these. Personal note-taking optimizes for organization, not learning. The Forte / Obsidian framing was always selling structure. There's nothing wrong with that. It's a different product.

Organizations generate all three natively. Decisions get revisited. Beliefs get revised. Patterns repeat across teams. Slack threads, code review cycles, design docs — every artifact a company produces is already shaped by recurrence and contradiction. The substrate has been there the whole time.

This is the move [Dorsey](https://block.xyz/inside/from-hierarchy-to-intelligence) was pointing at. The company world model isn't a copilot layered on top of hierarchy — it's the substrate that replaces what hierarchy used to do. Humans at the edge, where reality breaks the model. Closer to right than the rest of the field.

This is what we're building at Falconer. Org-scale memory where forgetting is the load-bearing primitive — supersession, schema formation, decay as a feature, not an accident.

The bigger piece on forgetting-as-the-key-to-learning is coming. Short version: if your second brain can only add, it isn't one.
